# Налаштування MCP для Cursor

У репозиторії [mcp.json](mcp.json) **без секретів**: сервери читають ключі з **оточення** процесу Cursor (або з вашого локального оновлення файлу).

## Варіант A — змінні оточення (рекомендовано)

Задайте в системі або в налаштуваннях Cursor (де підтримується):

| Змінна | Сервер |
|--------|--------|
| `CONTEXT7_API_KEY` | context7 |
| `GITHUB_PERSONAL_ACCESS_TOKEN` | github |
| `EXA_API_KEY` | exa |

Після перезапуску Cursor MCP підхоплять значення.

## Варіант B — копія з прикладу

1. Скопіюйте `mcp.example.json` → `mcp.json` (лише локально).
2. Підставте реальні ключі в блоки `env`.
3. **Не комітьте** `mcp.json`, якщо в ньому є справжні токени. Краще тримати секрети в `.env` / змінних оточення й не додавати файл із ключами в git.

Якщо ключі колись потрапили в історію git — **змініть (ротуйте)** їх у сервісах (GitHub, Context7, Exa).
